# トラブルシューティング

## サーバーにアクセスできない場合

### 1. サーバーが起動しているか確認

ターミナルで以下のコマンドを実行してください：

```bash
cd /Users/hayato/financial_rpg
python app.py
```

サーバーが起動すると、以下のようなメッセージが表示されます：

```
============================================================
金融知識学習RPG - サーバー起動中
============================================================
ポート: 8080

ブラウザで以下のURLにアクセスしてください:
  → http://localhost:8080
  → http://127.0.0.1:8080

サーバーを停止するには Ctrl+C を押してください
============================================================

 * Running on http://0.0.0.0:8080
 * Debug mode: on
```

### 2. 表示されたURLにアクセス

メッセージに表示されたURL（例: `http://localhost:8080`）をブラウザのアドレスバーに入力してください。

### 3. ポートが使用されている場合

エラーメッセージに「Address already in use」と表示される場合：

1. 他のアプリケーションがポートを使用している可能性があります
2. サーバーは自動的に別のポート（8080、3000など）を試します
3. 表示されたURLを使用してください

### 4. テストサーバーで確認

まずは簡単なテストサーバーで動作確認してください：

```bash
python test_server.py
```

ブラウザで `http://localhost:8080`（表示されたポート番号）にアクセスして、テストページが表示されるか確認してください。

### 5. よくある問題

#### 問題: "ModuleNotFoundError: No module named 'flask'"
**解決方法:**
```bash
pip install -r requirements.txt
```

#### 問題: "Permission denied" エラー
**解決方法:**
ポート番号を変更してください（1024以上のポートを使用）

#### 問題: ブラウザで "接続できません" と表示される
**解決方法:**
1. サーバーが実際に起動しているか確認（ターミナルでメッセージを確認）
2. ファイアウォールがブロックしていないか確認
3. 表示されたURLが正しいか確認（ポート番号に注意）

#### 問題: ページは表示されるが、ゲームが動作しない
**解決方法:**
1. ブラウザの開発者ツール（F12）でエラーを確認
2. コンソールにエラーメッセージがないか確認

### 6. 手動でポートを指定する場合

`app.py` の最後の部分を以下のように変更できます：

```python
if __name__ == '__main__':
    app.run(debug=True, host='0.0.0.0', port=8080, use_reloader=False)
```

### 7. ログの確認

サーバーを起動すると、アクセスログがターミナルに表示されます。エラーが発生している場合は、そのメッセージを確認してください。
